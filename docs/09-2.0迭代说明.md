# 慢慢说 2.0｜家庭共护迭代说明

## 迭代目标

1.0 解决“老人看不懂”，2.0 进一步解决“看懂以后怎么做、谁来确认”。

核心闭环：

`老人遇到问题 -> AI 解释与风险分级 -> 主动多问一步 -> 家属确认 -> 老人得到明确下一步`

## 本轮已完成

- 风险分级：高风险、需确认、暂未发现风险
- 判断依据：向老人解释 AI 为什么得出结论
- 主动关怀：提醒老人可能遗漏的下一个风险
- 容错表达：不要求老人一次说清，不因前后不一致责备用户
- 追问交互：主动询问“有没有点过链接”，根据回答提供不同的下一步
- 家属协作演示：发出求助后，老人可看到家属的明确反馈
- 语音提问：老人点击后直接说话，识别完自动分析，不要求打字
- 旧机降级：浏览器不支持语音时，清楚告知并保留文字与截图入口
- 官方知识检索：先检索医保、反诈和政务条目，再交给模型组织成适老语言
- 来源透明：结果页显示官方机构、原始链接和条目更新日期

## 人文关怀原则

- 不说“你怎么点了”，而是说“已经点过也没关系，我们先处理下一步”
- 不要求老人回忆完整过程，先停止可能继续扩大的风险
- 不只告诉“不要做什么”，还要告诉“现在可以做什么”
- 不把不确定包装成确定结论，信息不足时引导家属或官方确认

## 下一轮候选

1. 截图标注：直接标出不要点的链接或应该确认的按钮
2. 扩展官方知识：按城市补充医保、银行和政务的可核验流程
3. 真实家庭绑定：将当前演示升级为真实的家属通知与回复
