# 慢慢说｜技术方案

## 当前 Demo 技术栈

- Next.js、React、TypeScript
- CSS 响应式布局
- Web Speech API 语音播报
- Web Speech API 中文语音输入（浏览器支持时）
- Clipboard API 一键复制家属消息
- Canvas 图片端侧压缩
- 小米 MiMo 多模态接口
- DeepSeek 纯文字备用接口
- OpenAI Responses API 兼容保留
- PWA Manifest 与 Service Worker

当前版本已完成可配置的多模态模型接入。没有配置密钥、网络超时或模型异常时，会自动切换为安全演示结果，保证老人端和比赛路演都不会卡死。

## 当前实现

### 完整体验闭环

- 三个场景入口
- 截图、拍照或文字提问
- 真实分析接口与演示兜底
- 适老化结果卡片
- 三步行动建议
- 家属消息复制
- 路演模式自动播放

### 适老化交互

- 默认大字模式、高对比色彩和大按钮
- 一屏只做一件事，结果结构固定
- 支持语音播报
- 360px 窄屏无横向溢出
- 系统开启“减少动态效果”时自动降低动画
- 可添加到手机主屏幕，不强制从应用商店安装

## 模型分析链路

### 输入层

支持截图、拍照和手动输入问题。图片在浏览器内缩放到最长边 1600 像素并转为压缩 JPEG，再上传到服务端；服务端限制单张图片不超过 8 MB。

### 理解层

服务端会先根据问题类型和关键词，从本地结构化知识库中检索最相关的官方条目。当前包括国家医疗保障局、公安部和中国政府网来源。只有程序命中的条目会进入模型上下文，来源由服务端返回，不由模型自行生成。

服务端默认通过小米 `mimo-v2.5` 理解截图和文本：

- 判断短信、聊天、医保、银行或支付页面等场景
- 提取链接、验证码、手机号和官方名称等关键信息
- 识别陌生域名、索要验证码、诱导转账和冒充官方等风险

当 MiMo 暂时不可用且用户输入了文字时，服务端可切换到 DeepSeek 生成纯文字说明；DeepSeek 不接收图片，也不能把结果表述为图片识别结论。两个模型均不可用时使用安全演示模板。

### 决策与表达层

模型只返回固定结构：安抚语、短结论、关键提醒、通俗解释、最多三步行动和一段家属消息。服务端会再次清洗长度与枚举值，缺失字段使用安全模板补齐。

表达遵循四条规则：少术语、短句子、明确告诉老人不要做什么、每次最多三步。

## 隐私与安全边界

- 图片仅用于本次分析，当前产品不持久化保存
- 语音识别在浏览器能力层调用，本产品服务端只接收识别后的文字，不保存录音
- 官方知识条目保留来源链接和更新日期，地方政策不确定时必须建议人工核实
- 涉及陌生链接、转账、密码或验证码时默认采取谨慎策略
- 明确声明结果为辅助判断，不冒充医保、银行、公安等官方结论
- 网络或模型失败时不展示技术错误，切换到保守兜底建议

## 低端机适配策略

- H5 / PWA 优先，减少安装门槛
- 手机端只负责上传、显示和播放，模型推理在云端完成
- 图片上传前压缩，减少流量和等待时间
- 弱网状态显示明确等待提示
- 避免复杂动画、重资源和强硬件依赖
- 首页业务代码保持轻量
